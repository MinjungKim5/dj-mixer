# 자동 재생 모드 (Auto Mode) 기획

## 개요

덱 A와 B를 번갈아가며 자동으로 재생하고, 곡 전환 시 크로스페이드를 자동 수행하는 모드.
DJ가 수동 조작 없이도 끊김 없는 연속 재생이 가능하다.

## 전환 타임라인

```
덱 A 재생 중                                          덱 B 재생 중
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━┿━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
                                   -8s  -5s  -1s   0s
                                    │    │         │
                                    │    │         └─ A 종료 → A 다음곡 cue (start & stop으로 스탠바이)
                                    │    └─ 크로스페이드 시작 (4초간 A→B)
                                    └─ B 재생 시작 (프리롤 8초)
```

### 단계별 동작

| 시점 | 동작 |
|------|------|
| A 종료 8초 전 | B의 현재 큐 곡을 `play()` — 볼륨은 크로스페이더에 의해 아직 0 |
| A 종료 5초 전 | 크로스페이드 시작: 4초간 크로스페이더를 A→B 방향으로 이동 |
| A 종료 1초 전 | 크로스페이드 완료, B가 메인 출력 |
| A 종료 (0초) | A의 큐를 다음 곡으로 이동 → `loadVideo()` + 즉시 `pause()` (스탠바이) |
| B 종료 8초 전 | 같은 과정을 B→A로 반복 |

### 프로젝션 전환

크로스페이드 완료 시점(-1초)에 프로젝션 대상 덱을 전환 대상(B)으로 자동 변경한다.

## 시간 파라미터 (테스트 후 조정 예정)

```typescript
const AUTO_CONFIG = {
  prerollSec: 8,      // 다음 덱 사전 재생 시점 (종료 N초 전)
  fadeStartSec: 5,    // 크로스페이드 시작 시점 (종료 N초 전)
  fadeDurationSec: 4, // 크로스페이드 소요 시간
};
```

### 활성화 상태 표시

오토모드 활성화 시 **페이지 전체 테두리에 녹색 그라디언트 글로우**를 적용하여 한눈에 활성 상태를 인지할 수 있게 한다.

```css
/* 활성화 시 body 또는 최상위 div에 적용 */
border: 1px solid transparent;
background-clip: padding-box;
box-shadow: inset 0 0 0 1px rgba(34, 197, 94, 0.4),
            0 0 15px rgba(34, 197, 94, 0.15);
```

비활성 시에는 일반 테두리로 복귀.

## 상태 관리

```typescript
interface AutoModeState {
  enabled: boolean;
  activeDeck: DeckId;       // 현재 메인 출력 중인 덱
  phase: "idle" | "preroll" | "fading" | "standby";
}
```

- **idle**: 오토모드 비활성 또는 대기
- **preroll**: 다음 덱 사전 재생 중 (8초~5초 전)
- **fading**: 크로스페이드 진행 중 (5초~1초 전)
- **standby**: 전환 완료, 이전 덱이 다음 곡 스탠바이 상태

## 구현 시 고려사항

### 타이밍 감지

DjDeck에서 이미 250ms 간격으로 `getCurrentTime()`/`getDuration()`을 폴링하고 있으므로, 이 값을 page.tsx로 올려서 `remainingTime = duration - currentTime`으로 트리거 시점을 감지한다.

### 크로스페이드 애니메이션

CrossfaderCommand에서 구현된 `setInterval` + ease-in-out 애니메이션 로직을 재사용 가능. 오토모드에서는 page.tsx 레벨에서 `setCrossfader()`를 직접 제어해야 하므로 애니메이션 로직을 유틸로 분리하는 것이 좋다.

### YouTube Premium 필요

IFrame API는 브라우저의 YouTube 로그인 세션을 그대로 사용하므로, Premium 계정으로 로그인된 상태에서는 광고가 뜨지 않는다. 광고가 재생되면 크로스페이드 타이밍이 깨질 수 있으므로, 오토모드 사용 시 Premium 로그인 상태를 권장한다.

### 큐가 비었을 때

다음 곡이 없는 경우 오토모드를 자동 비활성화하고 현재 재생을 유지한다.
