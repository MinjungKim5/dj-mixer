# 슈퍼 심플 시트 큐(Playlist) 연동 구현서 (완료)

## 1. 개요
`super-simple-sheet.space`에서 제공하는 JSON 데이터를 활용하여 DJ Mixer의 Deck A, Deck B 큐(Queue)를 원격 재생목록과 주기적으로 동기화하여 관리하는 기능입니다.

## 2. UI 구성
- **위치**: 기존 "스마트 큐(Smart Cue)" 버튼과 같은 높이의 좌측 영역.
- **컴포넌트**: 연동할 재생목록의 이름을 입력할 수 있는 텍스트 인풋창 및 Sync 상태를 켜고 끄는 토글 버튼.
- 기능 활성화 시 버튼이 초록색 `SYNCING` 상태로 시각적 피드백을 제공합니다.

## 3. 데이터 연동 방식
- **API 엔드포인트**: `https://super-simple-sheet.space/json/{재생목록}`
- **CORS 및 캐시 우회**: Net.js Static Export 환경상 클라이언트에서 직접 페치하므로, 브라우저의 CORS 정책과 응답 캐시를 우회하기 위해 `corsproxy.io` 프록시를 경유하며, 매 요청마다 `?t={Date.now()}` 타임스탬프 파라미터와 `cache: "no-store"` 헤더를 추가합니다.
- **동기화 주기**: 15초마다 백그라운드에서 폴링(Polling)하여 최신 데이터를 페치합니다.
- **데이터 형식 매핑**:
  - **A열 (Column A)**: Deck A의 재생목록 대기열(Queue)
  - **B열 (Column B)**: Deck B의 재생목록 대기열(Queue)

## 4. 스마트 동기화(Smart Sync) 및 최적화 아키텍처

### 고민 1: 큐 업데이트 방식 - 증분(Incremental) vs 덮어쓰기(Overwrite)
시트를 단일 진실 공급원(Source of Truth)으로 취급하면서도, 로컬에서 사용자가 시청 완료한 내역이 유실되거나 화면이 무한 새로고침되는 현상을 철저히 방지해야 했습니다.

* **최종 구현 방식: 상태 보존 기반 스마트 덮어쓰기**
  1. **히스토리 보존**: 사용자 큐(Deck)에서 현재 장전되어 있거나 이미 지나간 트랙들(`historyQueue`)을 안전하게 잘라내고 보호합니다.
  2. **교집합(Diff) 탐색**: 새로 페치된 시트 큐(`fetchedQueue`)를 뒤에서부터 순회하며 로컬 `historyQueue`와 일치하는 마지막 `videoId`의 위치를 찾습니다.
  3. **재생 대기열 교체**: 교체 지점 이후의 시트 곡들만을 새로운 `upcomingQueue`로 산정하여 이전 히스토리의 꼬리에 결합합니다.

### 고민 2: 메타데이터(제목 & 썸네일) 유지 및 15초 렌더링 깜빡임 방지
로컬 환경에서는 곡이 추가될 때 유튜브 API에서 **실제 제목**과 **썸네일**을 받아오지만, 구글 시트의 JSON에는 해당 정보가 없거나(`A제목: ""`) `"Video"`로만 표시되어, 15초마다 새 데이터를 긁어오면 제목이 리셋되어 큐 전체가 갈아엎어지는 심각한 부작용이 있었습니다.

* **해결 방안: 메타데이터 상속(Inherit) 및 일괄 페치(Batch Fetching)**
  1. **데이터 상속**: 시트에서 긁어온 빈 정보가 담긴 `Track`들에 대해, **현재 덱(Deck)이 이미 보유하고 있는 곡(동일한 `videoId`)** 이라면 그 곡이 가지고 있던 기존 썸네일과 진짜 제목 정보를 그대로 복사옵니다.
  2. **병렬 일괄 조회**: 큐에 완전히 새롭게 추가된 곡(로컬에 보유 메타데이터가 없는 곡)이 있다면, 이들만의 `videoId`를 따로 추출(Set)합니다.
  3. **Batch Fetch API**: 유튜브 API(`fetchVideosMetadata`)에 배열로 넘겨 단일 요청(최대 50개)으로 필요한 메타데이터를 일괄 채워 넣음으로써 API Quota 낭비를 방지했습니다.
  4. **무결성 비교**: 상속 및 보충된 데이터 기반의 최종 결합 정보(`newQueue`)가 이전 상태(`prev.queue`)와 `JSON.stringify` 기준 일치할 경우 상태 변경 호출 자체를 중단(return prev)하여 극적인 퍼포먼스 안정성을 확보했습니다.

### 고민 3: 큐 업데이트가 현재 재생 중인 덱에 미치는 간섭 방지
  1. **상태 격리**: '현재 Deck에 장전된 트랙(Current Track)'과 '대기열(Queue)'을 분리된 상태(State)로 관리합니다. 대기열 배열이 수정되더라도 현재 재생 중인 트랙의 `currentIndex` 자체에는 변형을 가하지 않아 `DjDeck` 컴포넌트 내부에서 리렌더링되지 않습니다.
  2. **트랙 전환 매끄러움 유지**: 15초 단위의 인터벌 중에도 현재 재생곡이 변경되는 것을 지연시키거나 막지 않도록 React 함수형 State Updater 패턴의 인자로 안전한 시점의 Previous 값을 보장받도록 설계했습니다.
