# 큐(Playlist) 연동 구현서 - Super Simple Sheet 및 YouTube 재생목록

## 1. 개요
원격 데이터 소스(Super Simple Sheet 및 YouTube 재생목록)를 활용하여 DJ Mixer의 Deck A, Deck B 큐(Queue)를 주기적으로 동기화하여 관리하는 기능입니다.

## 2. 데이터 연동 방식

### A. Super Simple Sheet (구글 시트 기반)
- **API 엔드포인트**: `https://super-simple-sheet.space/json/{재생목록}`
- **동기화 주기**: 15초마다 백그라운드 폴링(Polling)
- **데이터 형식**: 시트의 A열데이터는 Deck A로, B열데이터는 Deck B로 각각 매핑됩니다.
- **특이사항**: 브라우저 캐시 방지를 위해 `?t={Date.now()}` 및 `cache: "no-store"`를 사용합니다.

### B. YouTube 재생목록 (YouTube API 기반)
- **API 호출**: YouTube Data API v3 (`fetchPlaylistTracks`)를 사용합니다.
- **동기화 주기**: 30초마다 백그라운드 폴링
- **분배 규칙 (Smart Queue Rule)**: 단일 재생목록을 가져와 **짝수 인덱스 트랙은 Deck A, 홀수 인덱스 트랙은 Deck B**로 자동 분배합니다.
- **UI**: 전용 입력창과 `YT SYNC` 버튼을 통해 활성화하며, 활성화 시 빨간색 애니메이션 피드백을 제공합니다.

---

## 3. 스마트 동기화(Smart Sync) 공통 아키텍처

두 동기화 방식 모두 `applySmartSync`라는 공통 로직을 공유하여 안정적인 큐 관리를 수행합니다.

### 핵심 로직: 상태 보존 기반 스마트 덮어쓰기
1. **히스토리 보호**: 현재 재생 중인 곡을 포함하여 이미 지나간 트랙들(`historyQueue`)은 절대 건드리지 않습니다.
2. **교집합(Diff) 탐색**: 새로 페치된 목록과 로컬의 히스토리를 비교하여, 마지막으로 일치하는 지점을 찾습니다.
3. **대기열 업데이트**: 일치 지점 이후의 새로운 곡들만을 `upcomingQueue`로 산정하여 기존 히스토리에 결합합니다.
4. **최적화**: 새로운 대기열이 기존과 완전히 동일할 경우(`JSON.stringify` 비교) React 상태 업데이트를 생략하여 불필요한 리렌더링을 방지합니다.

### 비동기 오류 및 Race Condition 방지
- **Active 플래그**: `useEffect` 내부에 `active` 플래그를 두어, 컴포넌트가 언마운트되거나 설정이 바뀌어 Effect가 재실행될 때 이전의 비동기 요청 결과가 현재 상태를 덮어쓰지 않도록 보호합니다.
- **Cleanup**: `clearTimeout`을 통해 이전 폴링 타이머를 확실히 제거하여 중복 실행을 방지합니다.

## 4. 메타데이터 최적화 (구글 시트 전용)

구글 시트 연동 시 발생하는 메타데이터 유실 문제를 해결하기 위해 다음과 같은 기법을 적용했습니다.

1. **메타데이터 상속(Inherit)**: 동일한 `videoId`를 가진 곡이 이미 덱에 존재한다면, 로컬에 저장된 실제 제목과 썸네일을 그대로 유지합니다.
2. **Batch Fetching**: 새로운 곡들에 대해서는 유튜브 API(`fetchVideosMetadata`)를 통해 최대 50개 단위로 한 번에 메타데이터를 보충하여 API 할당량 소모를 최소화합니다.
